## Vision

### 10x Check
The 10x version of this action is a deploy primitive that works across repos (charts in a
separate gitops repo), supports rollback (revert to previous tag), emits structured deploy
events consumed by status pages and Slack, and tracks deploy history per service/environment.
That's a full deploy platform, not a v1 action.

### Platonic Ideal (baseline)
A minimal, correct, observable action that handles the atomic-commit race condition, produces
informative PRs for gated environments, and fails with actionable error messages. An engineer
debugging a failed deploy should find the root cause from the action logs alone, without
needing to inspect git or GitHub API calls manually.

**3. Path containment check (deployer.go:valuesPath)**

`EvalSymlinks` fails on non-existent paths, so only call it on `ChartsDir` (which must exist
at action startup). For the constructed file path, use `filepath.Abs` only — the file may not
exist yet on first deploy:
```go
// valuesPath now returns (string, error) to allow the containment check to propagate.
absCharts, err := filepath.EvalSymlinks(filepath.Abs(opts.ChartsDir))
if err != nil {
    return "", fmt.Errorf("charts dir %q not accessible: %w", opts.ChartsDir, err)
}
absFile := filepath.Clean(filepath.Abs(result))
if !strings.HasPrefix(absFile, absCharts+string(os.PathSeparator)) {
    return "", fmt.Errorf("values path %q escapes charts dir", result)
}
```
Note: `filepath.Abs` on both sides normalizes `..` traversal without requiring the file to exist.

### Error quality

**4. Git push error discrimination (git.go:Push)**

Parse combined output after each failed push attempt. Match against known patterns:
- Auth failure: stderr contains `"403"`, `"unable to access"`, `"could not read Username"`, or `"Authentication failed"` → return `fmt.Errorf("authentication failed — check token has contents:write scope: %w", err)`, do not retry
- Push rejection (non-fast-forward): stderr contains `"rejected"` or `"non-fast-forward"` → retry with pull --rebase as currently implemented
- Other: propagate as-is

**5. GitHub API error body (github.go:do)**

On `resp.StatusCode >= 400`, read the body and attempt to decode `{"message":"..."}`:
```go
var ghErr struct{ Message string `json:"message"` }
body, _ := io.ReadAll(resp.Body)
json.Unmarshal(body, &ghErr)
if ghErr.Message != "" {
    return fmt.Errorf("http %s %s: status %d: %s", req.Method, req.URL, resp.StatusCode, ghErr.Message)
}
return fmt.Errorf("http %s %s: status %d", req.Method, req.URL, resp.StatusCode)
```

**8. Default branch detection (deployer.go:defaultBranch)**

```go
func defaultBranch(dir string) string {
    cmd := exec.Command("git", "symbolic-ref", "refs/remotes/origin/HEAD")
    cmd.Dir = dir
    out, err := cmd.Output()
    if err == nil {
        ref := strings.TrimSpace(string(out))
        if parts := strings.Split(ref, "/"); len(parts) > 0 {
            return parts[len(parts)-1]
        }
    }
    // GITHUB_DEFAULT_BRANCH is set on all event types (push, pull_request, etc.)
    // and reflects the repo's actual default branch. Available since 2021.
    // GITHUB_BASE_REF is NOT used — it's only set on pull_request events.
    if b := os.Getenv("GITHUB_DEFAULT_BRANCH"); b != "" {
        return b
    }
    return "main"
}
```

Note: `git symbolic-ref refs/remotes/origin/HEAD` may fail in shallow clones created with
`fetch-depth: 1`. The fallback to `GITHUB_DEFAULT_BRANCH` handles this for all event types.

### Accepted: Hybrid yaml.v3 approach (replaces values.go line scanner)

**HYBRID approach** (confirmed in eng review, ARCH-1): use yaml.v3 Node tree to FIND the target
node's line number, then perform byte-level line replacement on that specific line. This preserves
formatting exactly (no YAML reserialisation) while enabling correct YAML structure traversal.

SPEC.md explicitly requires: "No full YAML reserialise." yaml.v3 docs warn that re-encoded content
will not preserve original textual representation (defaults to 4-space indentation). Full round-trip
is therefore ruled out.

Approach:
1. Read file bytes + unmarshal into `*yaml.Node` (document node)
2. Traverse node tree to find the target scalar node (image tag, key path, or marker)
3. Record the target node's `Line` number (1-based)
4. Split file bytes into lines; replace only the target line(s) using byte-level manipulation
5. Write result back — formatting, comments, indentation unchanged

**Unified API (confirmed in eng review, ARCH-3):**
```go
// UpdateOptions selects the update strategy.
type UpdateOptions struct {
    Mode string // "image" (default) | "key" | "marker"
    Key  string // dot-notation path, for "key" mode only (e.g. "image.tag")
}

// SetTag finds the target node and updates its value.
// Returns (tagsReplaced, error). Returns error if the target is not found (ARCH-4).
func SetTag(file, tag string, opts UpdateOptions) (tagsReplaced int, err error)

// ReadTag reads the current value of the target node.
// Returns ("", nil) if not found (e.g. first deploy, file not wired yet).
func ReadTag(file string, opts UpdateOptions) (string, error)
```

Backward-compat: existing `SetImageTag` and `ReadImageTag` calls in deployer.go become
`SetTag(file, tag, UpdateOptions{Mode: "image"})` and `ReadTag(file, UpdateOptions{Mode: "image"})`.

The `gopkg.in/yaml.v3` dependency is already in `go.mod`. No new dependencies needed.

## Reviewer Concerns (3-iteration convergence)

These issues were raised by the adversarial reviewer and persisted after 2 revision rounds.
Use these as implementation checklists, not blockers.

1. **`ReadImageTag` signature not specified.** Implement as `ReadImageTag(path string) (string, error)` in `values.go`; return `("", nil)` when no `image:` block or no `tag:` is found. Same scanner logic as `replaceImageTag`, early return on first match.

2. **Old tag read timing.** Call `ReadImageTag(file)` BEFORE `gc.CheckoutBranch(...)`. The working tree has the current values file; reading after checkout would reset the file to the branch state (which may be different on a re-run). Step ordering in `deployPR()`: read old tag → checkout branch → SetImageTag → add → commit → force push → EnsurePR.

3. **`--force-with-lease` and shallow clones.** Document in README: callers must use `actions/checkout` with `fetch-depth: 0` for the PR deploy path, or the force-push may fail if the remote tracking ref is missing. CI workflow should include `fetch-depth: 0`.

4. **Step Summary file open flags.** Use `os.O_APPEND|os.O_WRONLY|os.O_CREATE` (not just `O_APPEND|O_WRONLY`).

5. **`git.Client.Add` already exists.** Confirmed in `git.go`. No new method needed.

6. **`valuesPath` signature change.** The auto-deploy path (`deployAuto()`) also calls `valuesPath()`. Update both call sites to handle the `(string, error)` return.

7. **Integration test remote.** Use a bare git repo (`git init --bare`) in a temp dir as the remote for integration tests. Set `git remote add origin <bare-dir>` on the working clone. No interface extraction needed; test against the real `git.Client`.
